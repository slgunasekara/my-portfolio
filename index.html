Variables

var · let · const

JavaScript-ල variables declare කරන්ට 3 ක්‍රම තිබෙනවා. Java ල int x = 10; කියලා type specify කරන්ට ඕනෙ, ඒත් JS ල type specify කරන්ට ඕනෙ නෑ — loosely typed language.

variables.js 
// Variable declare කිරීමේ ක්‍රම 3
var   a = 10;    // Non-block scoped (global/function scope)
let   b = 20;    // Block scoped — re-assign කරන්ට පුළුවන්
const c = 30;    // Block scoped — re-assign කරන්ට බෑ

// Scope demo
{
  var   a = 10;  // ✅ block ඉවත් වෙලා access කරන්ට පුළුවන්
  let   b = 20;  // ❌ block ඉවත් වෙලා access කරන්ට බෑ (ReferenceError)
  const c = 30;  // ❌ block ඉවත් වෙලා access කරන්ට බෑ (ReferenceError)
}
console.log(a); // 10 ✅
console.log(b); // ReferenceError ❌
console.log(c); // ReferenceError ❌

// const re-assign error
const x = 50;
x = 60; // ❌ TypeError: Assignment to constant variable
⚠️

Variable Naming Rules: Letters, _, $ දිය ගතහැකි. Number දිය ගත නොහැකි (e.g. 2myVar ❌). name ≠ Name (case-sensitive).

⚠️ Modern JS ල var avoid කරන්නේ ඇයි?
var use කරන්ට පුළුවන් — ඒත් 3 ලොකු problems තිබෙනවා
avoid var

Problem 1 — Scope Leak (Bug හදනවා)

scope-problem.js
// var — block ඉවත්ව access වෙනවා (unexpected!)
{
  var x = 10;
}
console.log(x); // 10 ✅ — block ඉවතිනුත් access වෙනවා!

// let — block scope safe ✅
{
  let y = 10;
}
console.log(y); // ❌ ReferenceError — expected behaviour

Problem 2 — Silent Re-declare (Bug detect කරන්ට බෑ)

redeclare-problem.js
var name = "Kavindu";
var name = "Ravindu"; // ✅ Error නෑ — silently overwrite! Dangerous!

let name2 = "Kavindu";
let name2 = "Ravindu"; // ❌ SyntaxError — caught immediately ✅

Problem 3 — Hoisting (Confusing behaviour)

hoisting-problem.js
// var — declare කරන්ඩෙ කලිනෙ use කරන්ට පුළුවන් (hoisting)
console.log(a); // undefined — error නෑ, ඒත් weird!
var a = 10;

// let/const — use කරන්ඩෙ කලිනෙ error ✅
console.log(b); // ❌ ReferenceError — sensible!
let b = 10;
✅

Modern Rule — ඕනම project ල follow කරන්න:

modern-rule.js
// ✅ Modern way
const PI      = 3.14;    // change නොවෙන values
let   count  = 0;      // change වෙන values
count++;                 // ✅ re-assign OK

// ❌ Avoid
var x = 10; // avoid in modern code

// 💡 IJSE class ල var use කරන්නේ ඉගෙනගන්ට පහසුවට.
//    Real projects ල let / const use කරනවා.

Comparison Operators
== vs === — Java ල නෑ මේ difference!
== === != !== > <
🔴

== vs === — == value compare කරනවා (type convert කරලා). === value සහ type දෙකම check කරනවා. Always === use කරන්න!

comparison.js
var a = 10, b = 10, d = "10";

a ==  d   // true  (value same, type ignore)
a === d   // false (number vs string — type different)
a === b   // true  (same value + same type)

a !=  d   // false (value same)
a !== d   // true  (type different)

a >  6    // true
a <  6    // false
a >= 10   // true
a <= 10   // true
Logical Operators
AND, OR, NOT
&& || !
logical.js
// AND (&&) — දෙකම true නම් true
true  && true   // true
true  && false  // false
false && false  // false

// OR (||) — එකක් true නම් true
true  || false  // true
false || false  // false

// NOT (!) — reverse
!true           // false
!false          // true
Unary Operators
++, --, unary +, unary -
++ -- +x -x
ℹ️

Pre vs Post: ++x —先 increment කරලා use කරනවා. x++ — use කරලා increment කරනවා.

unary.js
var a = "10";
console.log(a + 2);   // "102" (string concatenation!)
console.log(+a + 2); // 12 (unary + → number convert)

var x = 4;
// Pre-increment
var sum = 5 + ++x;  // x=5, sum=10
// Post-increment
x = 4;
sum = 5 + x++;       // sum=9, x=5
// Pre-decrement
x = 4;
sum = 5 + --x;  // x=3, sum=8
// Post-decrement
x = 4;
sum = 5 + x--;       // sum=9, x=3

Functions

Named · Expression · Arrow · Constructor

JavaScript ල functions declare කරන්ට 4 ක්‍රම තිබෙනවා. Arrow functions ES6 දී introduce වුණා — modern code ල ඒවාම use කරනවා.

1. Named Function
Most common — hoisted (declare කරන්ඩෙ ඉස්සෙල්ලත් call කරන්ට පුළුවන්)

3. Arrow Function (ES6)
Modern, concise — callbacks ල most used
() =>
💡

Single line arrow functions ල return keyword සහ {} ගහන්ට ඕනෙ නෑ!

Fetch API

Promise · .then() · JSON

Fetch API use කිරීමෙන් external APIs ලෙස data get කරන්ට පුළුවන්. Returns a Promise.

fetch-api.js
// Class example: JSONPlaceholder posts
fetch("https://jsonplaceholder.typicode.com/posts")
  .then((response) => response.json())  // JSON parse
  .then((data) => {
    data.map((item, index) => {
      $("#posts").append(`
        <div class="card">
          <h5>${item.title}</h5>
          <p>${item.body}</p>
        </div>
      `);
    });
  });

// Flow:
// 1. fetch() → sends HTTP GET request
// 2. .then(res => res.json()) → Response object → JSON parse
// 3. .then(data => ...) → data (array/object) ලෙස use
